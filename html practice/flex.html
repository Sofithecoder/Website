Flex Practice

display flex ბრძანება, მოქმედებს html ელემენტის შინაგან სამყაროზე. მაგრამ თუ გვინდა flex container გარეგან სამყაროზე ვიმოქმედოთ, მაგალითად ორი ასეთი კონტეინერი გვერდი-გვერდ დავაყენოთ, უნდა გამოვიყენოთ inline-flex ბრძანება.

FLEX DIRECTION არის ერთ-ერთი ბრძანება, რომელიც კონტეინერის შვილებს ეუბნება ერთმანეთთან მიმართებაში როგორ განლაგდნენ. row, row-reverse, column & column-reverse ღირებულებები აქვს

Flex wrap default არის no wrap. მაგრამ თუ გვინდა რომ შვილები, რომლებიც რეალურად ერთ ხაზზე არ ეტევიან თითოეულის ზომის გამო, დავტიოთ ერთ ხაზზე, უნდა გამოვიყენოთ flex wrap property.

Flex flow არის ორი ფროფერთის ჯამი flex-direction and flex-wrap. პირველი იწერება flex-direction და მეორე ვრეპი

შვილი ელემენტების თანმიმდევრობაზე რომ ვიმოქმედოთ, შვილ ელემენტებს order უნდა გავუწეროთ. სტანდარტულად გაწერილი აქვს 0. ამიტომ -1 იქნება მარცხნივ, 0 შუაში და 1 მარჯვნივ.

Justify content მუშაობს ჰორიზონტალურ ღერძზე და დაშორებებს აგვარებს შვილ ელემენტებს შორის. უნდა ჩაიწეროს მშობელი ელემენტის დეკლარაციაში

Align items property არის ვერტიკალურ ღერძზე ელემენტების განლაგების წესი. მშობელ კონტეინერს ვუწერთ და არა შვილებს. მნიშვნელობებია: stretch, flex-start, flex-end, center, baseline, auto

თუ გინდა კონტეინერის კონკრეტულ შვილებზე იმოქმედო იგივე მნიშვნელობებით რაც align items გააჩნია, მაშინ უნდა გამოიყენო align-self

Align content არის ფროფერთი, რომელიც ელემენტებს შორის დაშორებაზე ზემოქმედებს. flex-start, flex-end, stretch, center, space-bwtween, space-around

თუ flex-direction გვაქვს row or row-reverse მაშინ main axis არის ჰორიზონტალური ღერძი. თუ flex-direction გვაქვს column , column-reverse მაშინ main axis არის ვერტიკალური ღერძი

Flexibility

Flex basis property-ს დახმარებით, მშობელ ელემენტს ვეუბნებით, რომ მისი შვილები იდეალურ შემთხვევაში უნდა იყვნენ კონკრეტული ზომის.

Flex grow & flex shrink გვეუბნება, რომ როდესაც შენს კონტეინერს აღარ ექნება საკმარისი ადგილი ყველა ელემენტის მითითებულ ზომაზე დასასმელად, მაშინ როგორ უნდა გაზარდოს ან დააპატარავოს ელემენტები.

css property - FLEX აერთიანებს flex grow, flex-shrink

Flex grow სტანდარტული მნიშვნელობა არის 0 ანუ რო გაზარდო ბრაუზერი, ელემენტები არ გაიზრდება. მაგრამ flex shrink სტანდარტული მნიშვნელობა არის 1. რომ დააპატარავო კონტეინერი ყველა ერთნაირად დაპატარავდება. თუ flex shrink-ს ნულზე გადავიყვანთ, ელემენტები არ დაპატარავდება, უბრალოდ გადაიფარება
